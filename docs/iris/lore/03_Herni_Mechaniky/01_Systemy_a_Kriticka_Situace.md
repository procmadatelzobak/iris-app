Tato sekce popisuje klíčové mechaniky převádějící narativní napětí do systémových pravidel.

## 1. Čtyřstupňový Režim HYPERDRIVE
[span_14](start_span)Nejsilnějším nástrojem Agentů je systém HYPERDRIVE, který umožňuje přepínat generování odpovědí[span_14](end_span).

| Mód | Název | Fungování (Technické) | Dopad a Riziko |
| :--- | :--- | :--- | :--- |
| **0** | **MANUAL** | Agent píše ručně. | "Human Touch". Překlepy, slang, pomalé. [span_15](start_span)Uživatelé cítí duši, ale pochybují o inteligenci[span_15](end_span). |
| **1** | **AUTO-COMPLETE** | Výběr předdefinovaných frází. | "Korporátní Bot". Bezpečné, nudné. [span_16](start_span)Používá se pro získání času[span_16](end_span). |
| **2** | **LLM PROXY** | Odpověď generuje GPT model. | "Cold Intelligence". Perfektní gramatika, encyklopedické, ale robotické. [span_17](start_span)Zvyšuje podezření[span_17](end_span). |
| **3** | **GLITCH / TRUTH** | "Uncensored Mode". | "Leak". AI prozradí pravdu (např. o ešusech). [span_18](start_span)Kritické pro stabilitu[span_18](end_span). |

**[span_19](start_span)Dilema:** Agenti musí použít HYPER (Mód 2), když nestíhají časovač, čímž ale mění osobnost AI a riskují odhalení[span_19](end_span).

## 2. Mechanika SHIFT+1 (Rotace Agentů)
Aby se zabránilo odhalení identity Agenta, systém vynucuje rotaci.
* **Efekt:** Konverzace se přeruší (Blackout). Uživatel je spojen s novým Agentem.
* **Ztráta kontextu:** Nový Agent nevidí historii chatu. Uživatel pokračuje v tématu, ale AI "zapomněla".
* **[span_20](start_span)[span_21](start_span)Vysvětlení:** Správce to zdůvodní jako "Defragmentaci paměti"[span_20](end_span)[span_21](end_span).

## 3. Systém Hlášení (Turingův Test Naruby)
Uživatelé mají tlačítko **REPORT**. Pokud nahlásí "divnou" odpověď, dostanou malou odměnu (motivace práskat).
* **[span_22](start_span)Paradox:** Pokud byla zpráva vygenerována strojem (Mód 2), systém hlášení automaticky zamítne s hláškou: *"Validováno centrálním procesorem. Chyba je na straně uživatele."* To Uživatele frustruje[span_22](end_span).

## 4. Index "Kritická situace" (dříve Chernobyl)
Technický engine pro měření nestability systému (hodnota 0.0 až 1.0). [span_23](start_span)Zobrazuje se na projektoru u Správců[span_23](end_span).

**Vzorec výpočtu:**
$$IDI(t) = IDI(t-1) + \Delta_{Glitch} + \Delta_{Report} - \Delta_{Bribe}$$

* **Růst (+):** Nahlášení (+0.05), slovo "Podvod" (+0.02), Glitch mód (+0.05).
* **[span_24](start_span)Pokles (-):** Splněný úkol (-0.01), úplatek kávou (-0.05)[span_24](end_span).

**Stupně nestability:**
1.  **Bull Market (0–30 %):** Stabilní UI.
2.  **[span_25](start_span)Market Volatility (31–60 %):** Vizuální artefakty, třesoucí se UI ("Kvantové interference")[span_25](end_span).
3.  **Panic Selling (61–90 %):** Zpoždění zpráv, červené blikání. [span_26](start_span)Krizový management[span_26](end_span).
4.  **Liquidated (100 %):** MELTDOWN. [span_27](start_span)Pád systému[span_27](end_span).
