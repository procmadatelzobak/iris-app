Uživatelské terminály (Subjekty) – funkce a chování

Terminál hráče je retro konzole napojená na jeho přiřazeného Agenta. Následující výčet popisuje, co se děje v každé funkci – zobrazení stavů, chat, úkoly, ekonomika i blokace.

1) Přihlášení a směrování

  • Login obrazovka: Uživatel zadá jméno/heslo a je směrován do terminálu ROLE=USER. V testovacím módu se tlačítka doplní automaticky.
  • Směrování na session: Po přihlášení je hráč mapován na jedno ze 8 session ID. Agenti vidí toto ID a směrovací posun Shift určuje, kdo s kým mluví.

2) Chat a hlášení

  • Posílání zpráv: Zpráva se odešle přes WebSocket do přiděleného agenta. Historie se okamžitě doplní, pokud je viditelnost nastavena na NORMAL.
  • Indikace psaní: Při psaní se broadcastuje status typing, aby agent viděl živý vstup.
  • Report: Každý přijatý text lze nahlásit (REPORT). Pokud agent použil Optimizer, zpráva je chráněna před penalizací.
  • Glitch při přetížení: Pokud systém hlásí overload (teplota nebo výkon), UI se třese a převrací barvy.

3) Ekonomika a status

  • Zůstatek kreditů: Panel nahoře ukazuje aktuální bilanci. Při poklesu pod 0 se aktivuje Purgatory.
  • Status Level: Barva a písmo terminálu se mění podle úrovně L/M/H/P, kterou nastavuje admin (luxus, party, atd.).
  • Zdanění: Každá platba úkolu se zdaní sazbou Treasury (výchozí 20 %), která se připíše do pokladny.

4) Úkoly

  • Žádost o úkol: Tlačítko „Request New Task“ odešle požadavek admin stanici. Stav se nastaví na Pending.
  • Přidělení a úpravy: Admin může popis přepsat a nastavit odměnu podle status levelu. Po schválení se stav změní na Active a popis se ukáže v terminálu.
  • Odevzdání: Uživatel odešle text; admin jej ohodnotí 0–100 %, zdaní a připíše kredit.
  • Historie: Po zaplacení se úkol označí Completed/Paid a zůstává viditelný v historii session.

5) Purgatory (dluhový zámek)

  • Spouštěč: Bilance < 0. Chat panel se překryje červeným overlayem „COMMUNICATION OFFLINE“.
  • Chování: Chat je blokovaný, ale žádost a odevzdání úkolu stále fungují, aby hráč mohl dluh splatit.
  • Ukončení: Jakmile bilance >= 0, overlay zmizí a chat se odblokuje automaticky.

6) Dokumenty a lokalizace

  • In-app manuály: V menu je odkaz na Markdown viewer; styl se přizpůsobí roli (uživatel dostane základní vzhled).
  • Čeština: Veškeré popisky UI i systémové zprávy jsou přeložené, slovník lze přepsat custom labely z Admin HUBu.
