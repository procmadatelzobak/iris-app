Agentní terminály – nástroje a režimy

Agenti obsluhují relaci uživatelů přes monochromatický terminál. Tato kapitola rozepisuje každou funkci včetně AI nástrojů a bezpečnostních omezení.

Obrazová reference terminálu

1) Směrování a identita

  • Session banner: V levém horním rohu je ID session a aktuální globální Shift. Při změně Shiftem (admin/root) se agent automaticky přemapuje na jiného uživatele.
  • Role-based redirect: Po loginu ROLE=AGENT je terminál vynucen s příslušnými CSS styly a černozeleným tématem.

2) Chat a viditelnost

  • Režimy zobrazení: NORMAL ukazuje celou historii, BLACKBOX skrývá minulost (agent reaguje naslepo), FORENSIC/EPHEMERAL jsou rezervované pro budoucí použití.
  • Zprávy: Odeslání textu jde přímo na session uživatele a do Admin logu. Indikátor psaní se posílá v reálném čase.
  • Timeouty: Pokud agent neodpoví v časovém okně (výchozí 120 s), systém pošle uživateli chybovou hlášku a agent dostane blokaci pro danou zprávu.

3) Message Optimizer

  • Workflow: Agent napíše zprávu a klikne na „Optimize“. Text se pošle do LLM s promptem Admin/Root. Agent vidí náhled, může potvrdit nebo odmítnout.
  • Lock: Během optimalizace je vstup uzamčen, aby se předešlo duplicitám.
  • Imunita: Potvrzená optimalizovaná zpráva má imunitu vůči REPORT od uživatele.

4) Autopilot

  • Zapnutí/Vypnutí: Přepínač v terminálu aktivuje LLM odpovědi místo agenta. Stav je viditelný v banneru (ON/OFF).
  • Kontext: Autopilot drží historii konverzace per session; při přemapování na jiného uživatele se kontext resetuje.
  • Model: Název modelu je řízen z Root panelu (Hyper/Autopilot LLM). Poskytovatel se v UI nemění, backend používá výchozí OpenRouter.

5) Časovač odpovědí

  • Progress bar: Žlutý pruh běží od okamžiku, kdy uživatel odešle zprávu. Limit je nastavitelný v Controls/ROOT.
  • Timeout efekt: Po překročení limitu se do chatu pošle systémová zpráva o vypršení a agent je označen za opožděného.

6) Nástroje pro úkoly

  • Odevzdání/práce: Agent vidí stav úkolu uživatele a může potvrdit přijetí zpráv spojených s úkoly, ale schvaluje je Admin HUB.
  • Omezení viditelnosti: V BLACKBOX režimu agent nevidí předchozí úkoly a reaguje pouze na aktuální prompt.
