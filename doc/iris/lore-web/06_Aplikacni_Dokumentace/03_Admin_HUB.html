Admin HUB – stanice a ovládací prvky

Admin má čtyři hlavní stanice. Každá karta níže popisuje konkrétní chování a vazby na backend (kredity, logy, směrování a úkoly).

Obrazová reference Admin HUBu

1) Monitor (Panopticon)

  • Overview: Rozdělený pohled Session Grid + Mini Log. Streamuje nové zprávy a stavy v reálném čase.
  • Chats tab: Osm panelů chatu (sessions 0–7). Každý panel zobrazuje uživatele/agent zprávy a umožňuje rychlou kontrolu konfliktů.
  • System Logs: Barevně kódované události (ACTION/ROOT/REPORT/TASK). Tlačítko „Reset Log“ smaže historii v databázi i UI.

2) Controls

  • Agency Operation Mode: Přepíná Normal/Low Power/Overclock. Ovlivňuje rychlost ochlazování teploty a spotřebu výkonu.
  • Visibility Protocols: Přepíná HyperVisibility (Normal/Blackbox/Forensic/Ephemeral) pro agentní terminály.
  • Temperature slider: Manuální override 0–350+. Hodnota se promítne do broadcastu a spouští glitch efekty.
  • Agent Response Timer: Nastavuje globální limit pro odpovědi agentů (výchozí 120 s). Timeout se sleduje v game loopu.
  • AI Optimizer: Toggle ON/OFF a úprava promptu. Aktivní stav zvyšuje spotřebu výkonu.
  • Shift Execution: Tlačítko ">> EXECUTE SHIFT >>" incrementuje globální offset směrování.

3) Economy

  • Grid uživatelů: Tabulka bilancí a status levelů. Každé tlačítko provede API volání s okamžitým websocket broadcastem.
  • Fine/Bonus: Přičítá nebo odečítá kredity. Při odečtu pod 0 se uživatel dostane do Purgatory.
  • Lock/Unlock: Ručně aktivuje/deaktivuje Purgatory (chat blok, úkoly povoleny).
  • Status Level Buttons: Přepíná vzhled terminálu (Low/Mid/High/Party) a předvyplňuje budoucí odměny úkolů.
  • Treasury display: Ukazuje pokladnu po zdanění úkolů; následující kroky v Controls snižují/zvyšují výkon.

4) Tasks

  • Pending queue: Zobrazuje žádosti „Request New Task“. Admin může přepsat text a nastavit custom reward.
  • Approve: Uloží schválený popis a odměnu; změní stav na Active a odešle do terminálu.
  • Pay: Otevře modální okno (prompt vlevo, odpověď vpravo). Admin vybere procenta 0/50/100/200, odečte daň a připíše kredit.
  • Reject: Zavře úkol s označením Rejected; hráč může žádat další.
  • History: Seznam dokončených úkolů pro audit, nesmaže se při resetu logu.

5) Síťové a globální akce

  • Global Broadcast: Odesílá zprávu všem session naráz (uživatelé i agenti). V logu se označí jako ROOT/ACTION.
  • System Reset (NUKE): Vymaže logy, kredity, úkoly a vrátí GameState do defaultu (teplota, shift, power, LLM prompty).
  • Force Shift: Rychlé přemapování agentů (inkrement offsetu). Aktualizuje graf sítí.
  • Network Graph: Canvas vizualizace vazeb User-Agent, aktualizuje se po každém shiftu.
