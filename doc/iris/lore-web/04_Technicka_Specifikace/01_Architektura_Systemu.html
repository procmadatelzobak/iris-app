TECHNICKÁ ARCHITEKTURA A UX

Technické řešení jako aktivní narativní prvek reflektující korporátní neschopnost.

1. Hardwarová infrastruktura: Princip ZED

ZED (Zero External Dependency) – nulová externí závislost.

  • Technické vysvětlení: Systém běží lokálně na jednom serveru (laptop v krabici) bez internetu pro eliminaci výpadků a latence.
  • Narativní vysvětlení: „Ochrana integrity kvantových stavů hliníku před dekoherencí z veřejného internetu."
  • Skutečný důvod: Nezaplacený internet a strach vedení, že Uživatelé vygooglí nesmyslnost „hliníkové paměti".

2. Softwarový stack: „Spaghetti Code"

  • Framework: Python 3.10+ a FastAPI („Fast" zní dobře).
  • Databáze: SQLite soubor; prezentováno jako „decentralizovaná atomická struktura".
  • Komunikace: WebSockets („Neuronová vlákna") pro real-time chat.

3. UX Design a Glitche

Retro-futuristický vzhled roku 2025 z pohledu roku 1990.

  • Barevná paleta: Pozadí černá (#000000); akcenty neon cyan (#00FFFF) a magenta (#FF00FF).
  • Písmo: Monospace (Courier New, Roboto Mono).

Visual Glitch Engine: Při růstu Kritické situace se UI třese, barvy se invertují, text se rozpadá. Uživatelům je vysvětleno jako „vizualizace intenzivního myšlenkového procesu AI".

4. Mechanika „Perfect Mirroring" (Dohled)

  • Funkce: Draft Sync – Správci a Agenti vidí text, který Uživatel teprve píše.
  • Korporátní newspeak: „Technologie prediktivní empatie."
  • Účel: Totální dohled; Správci mohou zasáhnout před odesláním nevhodných zpráv.
